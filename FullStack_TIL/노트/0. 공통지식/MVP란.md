**웹 서비스에서 MVP는 ‘Minimum Viable Product(최소 기능 제품)’을 뜻하며, 핵심 가치만 담아 빠르게 시장 반응을 검증하기 위한 전략적 접근 방식입니다. 즉, 완벽한 서비스가 아니라 고객이 실제로 원하는지 확인할 수 있는 최소한의 기능을 갖춘 초기 버전입니다.**

## 🔑 MVP의 핵심 개념

- **정의**: 고객에게 가치를 전달할 수 있는 최소한의 기능을 갖춘 제품 또는 서비스
- **목적**: 시장 반응을 빠르게 확인하고, 불필요한 자원 낭비를 줄이며, 아이디어의 성공 가능성을 검증
- **철학**: 완벽함보다 **빠른 검증과 학습**을 중시. 실패를 최소화하고 개선 방향을 빠르게 찾는 것에 초점

## 📊 MVP의 장점

|장점|설명|예시|
|---|---|---|
|**리스크 절감**|초기 투자 비용과 시간을 최소화|단순 웹 페이지로 고객 반응 확인|
|**빠른 시장 진입**|경쟁사보다 먼저 사용자 피드백 확보|Airbnb 초기 웹사이트 사례|
|**가설 검증**|고객 문제와 솔루션의 적합성 확인|“저렴한 숙박 공간 수요” 검증|
|**지속적 개선**|사용자 피드백 기반으로 기능 확장|카카오톡 초기 버전 → 다양한 기능 추가|

## 🚀 실제 적용 사례

- **Airbnb**: 단순한 웹사이트로 숙박 공간 공유 수요를 검증 후 확장
- **Dropbox**: 실제 제품 없이 시연 영상으로 고객 반응 확인 후 개발 진행
- **카카오톡**: 초기에는 단순 메시징 기능만 제공, 이후 피드백 기반으로 다양한 기능 추가
