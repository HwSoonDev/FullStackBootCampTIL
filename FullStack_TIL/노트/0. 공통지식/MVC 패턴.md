
MVC 패턴은 **Model-View-Controller**의 약자로, 애플리케이션을 세 가지 역할로 나누어 개발하는 아키텍처 패턴이에요. 핵심은 **관심사의 분리(Separation of Concerns)**입니다. 즉, 데이터 처리·비즈니스 로직과 화면(UI)을 분리해 유지보수성과 확장성을 높이는 것이 목적이에요.

구성 요소

- **Model (모델)**

- 데이터와 비즈니스 로직 담당
- DB와의 연동, 데이터 검증, 상태 관리 수행

- **View (뷰)**

- 사용자에게 보여지는 화면(UI) 담당
- 모델에서 받은 데이터를 시각적으로 표현

- **Controller (컨트롤러)**

- 사용자의 입력을 받아 모델과 뷰를 연결
- 흐름 제어, 요청 처리, 적절한 뷰 선택

동작 흐름 예시 (웹 애플리케이션 기준)

1. 사용자가 웹 브라우저에서 버튼 클릭 (입력)
2. **Controller**가 요청을 받아 **Model**에 데이터 처리 요청
3. **Model**이 DB에서 데이터를 가져와 결과 반환
4. **Controller**가 결과를 **View**에 전달
5. **View**가 화면에 결과를 표시

장점

- UI와 로직이 분리되어 유지보수 용이
- 여러 뷰에서 동일한 데이터를 재사용 가능
- 테스트 용이 (로직만 따로 검증 가능)

단점

- 구조가 복잡해져 작은 프로젝트에는 오히려 부담
- 각 구성 요소 간의 연결을 잘못 설계하면 오히려 관리가 어려워짐

👉 쉽게 비유하면, **Model은 요리 재료와 레시피**, **View는 손님에게 내놓는 음식**, **Controller는 요리사** 역할을 하는 셈이에요.
