SaaS는 **Software as a Service(서비스형 소프트웨어)**의 약자예요. 쉽게 말해, 소프트웨어를 직접 설치하지 않고 **인터넷을 통해 구독 형태로 사용하는 모델**을 뜻합니다.

핵심 개념

- **클라우드 기반 제공**: 소프트웨어가 클라우드 서버에 호스팅되어 웹 브라우저나 앱을 통해 접속
- **구독 요금제**: 사용자는 필요한 기간만큼 비용을 지불하고 사용
- **운영·업데이트 책임**: 서비스 제공자가 인프라 관리, 보안, 업데이트를 모두 담당

장점

- 설치와 유지보수 불필요 → 시간과 비용 절감
- 인터넷만 있으면 어디서든 접근 가능 → 높은 이동성
- 자동 업데이트 → 항상 최신 기능과 보안 유지

단점

- 인터넷 연결이 필수 → 네트워크 문제 시 사용 불가
- 데이터가 외부 서버에 저장 → 보안·개인정보 이슈 가능
- 장기적으로 구독 비용이 누적될 수 있음

대표적인 SaaS 서비스

- **Google Workspace** (Docs, Gmail, Drive)
- **Microsoft 365** (Word, Excel, Teams)
- **Slack** (팀 커뮤니케이션)
- **Dropbox** (파일 저장·공유)
- **Salesforce** (CRM)

오늘날 기업들은 평균적으로 수십~수백 개의 SaaS 애플리케이션을 사용하고 있으며, 글로벌 시장 규모는 2024년에 약 4천억 달러에 달했고 2030년까지 두 배 이상 성장할 것으로 예상돼요.

👉 정리하면, SaaS는 **“소프트웨어를 제품으로 사는 게 아니라 서비스로 빌려 쓰는 방식”**이라고 이해하면 가장 간단합니다.
