
Query의 중요 사용 이유는 **캐시** 때문이다.

캐시는 다시 사용될 것 같은 데이터를  임시로 저장해두는 공간을 말한다. 
**(브라우저 임시 저장 공간)**

Query는 queryFn(api)로 받은 데이터를 
queryKey를 통해 캐시에 저장하고, 
매번 다시 받아오지는 않는다.

서버 데이터가 바뀌지 않았을 가능성이 더 높기 때문이다.
대신 저장된 캐시를 사용한다.

사이트의 특징에 따라 
캐시 데이터를 사용하는 시간을 다르게 잡으면 된다는 것을 알 수 있다.

## Stale Time & gcTime
데이터가 상하는 시간을 (Stale Time) 설정하고
데이터를 더 이상 쓰지 않는 시간도(gcTime) 설정한다.

```
Stale Time: Stale = 상한 데이터 
			기본값 : 0 (서버에서 받자마자 상한 것으로 간주.)
gcTime: gc =  garbage collection 가비지 컬랙션.
		기본값 : 5분 (5분 동안 사용되지 않으면 캐시 영역에서 해제(제거_))
```

[[Stale (상한 데이터)]]

예시 코드
```jsx
function HomePage() {
  const result = useQuery({
    queryKey: ['posts'],
    queryFn: getPosts,
    staleTime: 60 * 1000,
    gcTime: 60 * 1000 * 10,
  });
  console.log(result);

  return <div>홈페이지</div>;
}
```
