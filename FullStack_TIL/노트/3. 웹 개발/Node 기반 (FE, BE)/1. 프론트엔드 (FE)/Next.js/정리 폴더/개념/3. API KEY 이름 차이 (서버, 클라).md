```
`CAT_API_KEY` => 서버 사이드  => 서버 컴포넌트
`NEXT_PUBLIC_CAT_API_KEY` = 클라이언트 사이드 => 클라이언트 컴포넌트
```

.env 파일을 작성할 떄,
서버 사이드 api key와 클라이언트 사이드 api key를 분리해서 써야 한다고 한다.

클라이언트 사이드 KEY를 따로 생각하는 이유는
브라우저에서 접근하기 위해서는 `NEXT_PUBLIC`이 앞에 붙어있어야 한다는 차이가 있기 때문이다.
`NEXT_PUBLIC`이 앞에 없으면 브라우저에서 .env의 키에 접근 할 수 없다.
즉, **클라이언트 컴포넌트**라면 이 KEY(환경변수)에 접근할 수 없다.

하지만 **서버 컴포넌트**라면, 그 외의 이름이라도 접근할 수 있다.

| 서버 사이드 KEY                    | 클라 사이드 KEY       |
| ----------------------------- | ---------------- |
| 민감한 정보 -> 절대 노출 금지            | 공개 가능 (제한된 권한)   |
| 예시)<br>- oauth API 키, ID <br> | 예시)<br>- 백엔드 URL |
