
Next.js는 4가지의 렌더링 방식을 지원한다,
정확히는 page router 방식 시절의 용어이지만, 동작 원리를 이해하기 위해 좋은 개념적 접근이라서 이 4가지로 나눠 생각한다.

참고) [[Route Segment Config]] - 각 라우트 경로의 동작(렌더링 방식) 제어 문법

## SSG - 정적 사이트 렌더링

Static Site Generation

빌드 시에 미리 html 페이지를 만들어 사용자에게 제공하는 방식.
Next.js의 `기본 렌더링 방식`이다.
`빌드 시` 해당 컴포넌트를 `100% 프리렌더링`하여 요청 시 제공합니다.

## ISR - 요청에 따른 정적 재렌더링

Incremental Static Regeneration

SSG의 확장판으로, 특정 페이지를 요청할 때만 새로 빌드해 캐시에 저장하는 방식.
빌드시 `100% 프리렌더링`하고 일정 시간이 지나면 사용자의 요청에 따라 페이지를 재생성할 준비를 한다.

페이지 스크립트에 realidate 변수를 설정해주면 ISR 방식으로 동작한다.
```js
// layout.tsx | page.tsx | route.ts 
// Next.js 13 버전부터 유효
export const revalidate = 5; // 5초 간격의 ISR


export default async function Page() {
  const res = await fetch('https://api.example.com/posts');
  const posts = await res.json();

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

## SSR - 서버 사이드 렌더링

Server-Side Rendering

사용자 요청 시마다 서버에서 컴포넌트 방식 렌더링을 하고, 완성된 html을 제공하는 방식.
빌드를 하는 것은 아니다.

장점
- SEO 최적화
- 빠른 렌더링 속도 (클라이언트 디바이스의 성능에 의존하지 X)

기본적으로 Next.js는 가능하면 SSG/ISR 방식으로 만들어 최적화를 시도하지만,
`dynamic` 변수를 설정해주면 강제로 서버 사이드 렌더링 (동적 제공 방식)으로 고정 할 수 있다.

```js
export const dynamic = 'force-dynamic';
```

```js
// layout.tsx | page.tsx | route.ts
// Next.js 13버전부터 유효
export const dynamic = 'force-dynamic';

export default async function Page() {
  const res = await fetch('https://api.example.com/data', {
    cache: 'no-store', // 항상 최신 데이터 가져오기
  });
  const data = await res.json();

  return <div>{data.message}</div>;
}

```

즉, params 같은 외부 요소로 인한 동적 렌더링이 들어가지 않는다면, 
기본적으로 정적 제공이 된다는 뜻이다.

SSR = params

## CSR - 클라이언트 사이드 렌더링

리액트 본연의 클라이언트 사이드 렌더링 방식이다.
브라우저에서 js 스크립트를 통해 컴포넌트 레벨에서 조합하고 페이지를 렌더링한다.

리액트에서는 모든 페이지가 CSR 방식이지만, Next.js에서는 명시적으로 `"use client"`를 적어주어야 한다. CSR적인 요소(리액트 훅: useState, useEffect,  커스텀 훅 등)를 사용하려고 하면, 클라이언트 컴포넌트로 명시하라는 오류가 나온다.

```js
"use client"
```

```js
"use client"

export default async function Page() {
  const res = await fetch('https://api.example.com/data', {
    cache: 'no-store', // 항상 최신 데이터 가져오기
  });
  const data = await res.json();

  return <div>{data.message}</div>;
}
```
