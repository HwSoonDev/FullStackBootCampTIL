프론트에서 백엔드 구현하기.
api 요청 처리를 app 라우터 방식 주소 폴더에 구성하는 것.
route.ts(js)는 page.js와 함께 둘 수 없다.

`route.ts` 또는 `route.js` 파일을 통해 특정 경로에 대한 HTTP 요청을 직접 처리할 수 있고,
백엔드의 동작을 따라할 수 있게 된다.

app router 방식으로 디렉토리 경로에 따라 api 주소가 매핑되고, GET, POST 등의 HTTP 형식을 사용할 수 있다. `next/server`의 `NextResponce`를 활용하여 api 리스폰스를 돌려준다.

```ts
import { NextResponse } from 'next/server' 

export async function GET() { 
	return NextResponse.json({ message: '안녕하세요!' }) 
} 

export async function POST(request: Request) { 
	const body = await request.json() 
	return NextResponse.json({ 받은_데이터: body }) 
}
```  

- `app` 디렉토리 내부에 `route.ts` 파일을 생성
- 경로는 디렉토리 구조에 따라 자동으로 매핑
- 예: `app/api/user/[id]/route.ts` → `/api/user/123` 같은 동적 라우팅도 가능

다음과 같은 메서드를 각각 함수로 정의:

- `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `OPTIONS`

각 메서드는 독립적으로 관리되므로 코드가 더 깔끔하고 유지보수가 쉽다.
