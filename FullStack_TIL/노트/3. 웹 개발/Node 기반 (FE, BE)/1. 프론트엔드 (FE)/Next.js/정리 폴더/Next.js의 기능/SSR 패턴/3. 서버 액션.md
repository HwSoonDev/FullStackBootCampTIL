
서버 액션이란
Form 태그의 action에 함수를 연결하여 서버 컴포넌트에서 폼 제출을 할 수 있게 한 것.
클라이언트 컴포넌트와 같은 클라 상태 관리가 없어도 동작할 수 있는 것이 특징.

```jsx
export default function Page() {
  async function createInvoice(formData) {
    'use server'
 
    const rawFormData = {
      customerId: formData.get('customerId'),
      amount: formData.get('amount'),
      status: formData.get('status'),
    }
 
    // mutate data(예시)
    await db.invoice.create({
			data: rawFormData,
		})
		
    // revalidate cache(예시)
    revalidatePath('/invoices') // 혹은
		revalidateTag('invoice-list')
  }
 
  return <form action={createInvoice}>...</form>
}
```

action에 들어가는 함수는 인자로 formData(폼의 데이터)를 받고, .get()함수로 Input 값에 접근이 가능하다. 간단하게 `서버 액션 함수`라고 부른다.

중요한 점은 `서버 액션 함수`는 `'use sever'`를 처음에 적어 주어야 한다는 점이다.

### 서버액션 장점

- 보안 : api 요청 시 네트워크에 민감한 정보가 노출되지 않고 실행. 브라우저의 보안 위협에서 안전.
- 개발자 경험 : 별도의 api 엔드포인트를 관리할 필요 없음
- 번들최적화 : 서버액션 코드는 JS 번들파일에 포함되지 않음

### 서버액션 단점

- 보안에 민감한 정보를 다루지 않고, 실시간 유효성 검사가 필요한 경우라면 폼 제출 시 서버액션은 좋은 선택지가 아닐 수 있다.
- `클라이언트 → Next.js 서버 → API 서버` 로 중간에 Next.js 서버를 거쳐 api 레이턴시가 오히려 더 늘어나 UX 저하의 원인이 될 수 있다.
- Tanstack Query 같은 전역 상태 관리, 프로바이더를 이용할 수는 없다. Tanstack Query도 훅이기 때문에 서버 컴포넌트에서는 사용할 수 없다.
