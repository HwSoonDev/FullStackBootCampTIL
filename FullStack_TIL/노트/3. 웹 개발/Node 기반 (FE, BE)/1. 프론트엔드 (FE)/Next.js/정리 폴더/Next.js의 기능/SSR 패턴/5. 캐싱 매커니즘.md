### 1_ 첫 번째 요청 : /a 페이지에 처음 들어온 경우

1. Route cache를 확인해요
    - Next.js는 가장 먼저 “이 페이지를 먼저 방문했었나?”를 Route Cache에서 찾아봐요
    - `MISS`인 경우(방문한 적이 없다면), 다음 단계로 넘어가요
2. Full Route Cache
    - 이전에 이 페이지가 SSG(또는 ISR)처럼 빌드 타임에 미리 만들어진 적이 있는지 확인해요
    - `HIT`인 경우 `RSC Payload`와 `HTML`을 읽어서 그대로 응답하기 때문에 `빨라요`
    - 없다면 렌더링을 새로 시작해요

### 2_ 렌더링을 (새로) 시작해야 하는 경우

1. 데이터 요청 시작(fetch 호출)
    - 예를 들어, **await fetch(’api/invoice’)**처럼 데이터 요청이 일어남
2. Request Memoization 확인
    - 혹시 이전에 계산된 것이 있는지 메모리에서 검사
    - 즉, 이미 요청해서 반환했던 **`함수의 결과값`**을 가지고 있는지를 검사하는 것
    - `MISS`인 경우, 새롭게 실행 후 `결과값`을 `SET`
3. Data Cache 확인
    - fetch의 결과가 JSON으로 캐싱이 되어있는지 확인
    - `MISS`인 경우, 원본 서버(API, DB 등)로 가서 실제 데이터를 가져옴
    - Data Source `HIT` (MISS 일 수는 없겠죠?)
    - 그리고, **Data Cache**에 저장(`SET`)

### 3_ 렌더링 결과 저장

1. 페이지를 서버에서 렌더링함
    - 렌더링 한다는 것은? **HTML + RSC payload 생성**한다는 것
    - 이렇게 만든 결과
        - Full Route Cache에 저장(디스크) : 다음 방문 시 빠르게 사용
        - Route Cache에도 저장(메모리) : 즉시 전환할 때 빠름

### 4_ 사용자에게 응답

1. HTML + RSC Payload를 사용자에게 보냄
2. 처음은 느릴 수 있지만, 다음 방문부터는 대부분 캐시 `HIT`로 빨라짐

## 용어 정리

**Router Cache** : 클라이언트 내에 RSC Payload(화면렌더링에 필요한 정보)를 저장
- RSC Payload란, 자바스크립트 오브젝트로 되어있고 메모리에 저장되어 빠르게 페이지 전환을 가능케 하는 데이터
 - 페이지 방문 시 Cache `SET`, 페이지 재방문 시 Cache `HIT`
 - SSR 페이지도 Cache HIT 됩니다! (no-store 옵션은 Router Cache 도 사용하지 않습니다.)
- 브라우저 세션동안만 유지되는 캐시 (브라우저창 닫거나, 새로고침 시 초기화)

**Full Route Cache** : 서버 컴포넌트 렌더링 결과인 html, RSC Payload가 저장되는 공간
    → Full Route Cache `SET` 되는 경우
- SSG
- ISR
- generateStaticParams

**Request Memoization** : 같은 렌더링 과정에서 발생하는 fetch 요청을 일시적으로 캐싱하는 공간
    
예시 코드
```jsx
// 같은 렌더링 사이클 내에서 동일한 fetch 호출
async function Page() {
  // 첫 번째 fetch - 실제 네트워크 요청 발생
  const data1 = await fetch('https://api.example.com/data', { cache: 'force-cache' });
  
  // 두 번째 동일한 fetch - request memoization HIT!
  const data2 = await fetch('https://api.example.com/data', { cache: 'force-cache' });
  
  // 두 변수는 동일한 데이터를 참조
  const result1 = await data1.json();
  const result2 = await data2.json();
  
  // ...
}
```

fetch api 를 사용하지 않더라도 Request Memoization 사용하는 방법
(→  react 의 cache 를 사용한다.)
```jsx
import { cache } from 'react'

// 사용자 정의 함수 메모이제이션
export const getUser = cache(async (id) => {
  // 비용이 많이 드는 연산 또는 데이터 처리
  const user = await db.query(...);
  return transformUserData(user);
})
```

**Data Cache** : fetch 로 외부 데이터 소스(API, DB 등)로부터 가져온 데이터를 캐싱하는 공간
```js
fetch(”url”, { cache: “force-cache” })
```

서버 컴포넌트 실행 시 fetch 할 때 no-store 캐시옵션이 없으면 Data Cache 에 SET 적용됨


### - 캐시 무효화 방법

- `revalidatePath()` - 특정 경로의 모든 캐시 레이어 무효화
- `revalidateTag()` - 특정 태그가 붙은 데이터 캐시만 무효화
- `router.refresh()` - 클라이언트에서 현재 경로 새로고침 (서버 컴포넌트 재실행)
