matadata는 SEO(검색 엔진 최적화)를 위해 입력한다.

matadata는 **(1)루트 Layout** 에서 설정이 가능하고, **(2)페이지 별**로도 설정이 가능한데,

정적 설정과 동적 설정이 있다.



### 정적 metadata

흔히 사용하는 정적 설정은 간단하다.

```jsx
export const metadata = {
  title: '...',
  description: '...',
}
 
export default function Page() {}
```

`export const metadata`의 형식으로 레이아웃이나 페이지 첫 단에 적어주면 된다.


### 동적 metadata

부모의 metadata를 자식 컴포넌트에서 쓰는 등의 방식도 가능한데,
여기서는 부모 컴포넌트의 스크립트에서 선언한 metadata를 자식 컴포넌트의 파라미터로 넘겨주고 있다.
```jsx
export async function generateMetadata({ params, searchParams }, parent) {
  // read route params
  const { id } = await params
 
  // fetch data
  const product = await fetch(`https://.../${id}`).then((res) => res.json())
 
  // optionally access and extend (rather than replace) parent metadata
  const previousImages = (await parent).openGraph?.images || []
 
  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}
 
export default function BreedsPage({ params, searchParams }) {}
```
