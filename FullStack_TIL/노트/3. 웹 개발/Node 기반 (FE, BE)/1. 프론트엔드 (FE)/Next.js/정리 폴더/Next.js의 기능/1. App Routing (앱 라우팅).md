## 앱 라우팅

프로젝트의 폴더 구조를 사이트 주소 경로에 그대로 사용하는 방식.
따로 router 경로를 코드 상으로 지정해주지 않아서 편하다 (page routing 방식과 비교)

![[Pasted image 20251226180741.png]]

## 동적 앱 라우팅

### `[폴더명]` 

: 이어지는 경로의 이름과 상관 없이 해당 페이지를 보여준다. 폴더 이름은 경로의 파라미터 이름으로 취급된다. 

예시 => `/products/[id]` `/products/efdjfakke22jkfe`  상품 아이디에 해당하는 정보를 페이지에 표시
### `(폴더명)`

: 해당 폴더는 경로(path)에 포함 시키지 않는다. 하지만 하위 폴더는 경로에 이어서 들어간다.

보통 유저에게 보이지 않는 경로 분류를 만들 때 사용한다.

예시 => `/(dev)/user/my` 실제 브라우저에서는 `/user/my`로 보인다.

### `_폴더명`

: 하위 폴더까지 포함시키지 않는다. 보통 개발용이나, 페이지 하위 컴포넌트를 넣어두는 용도로 사용된다. 하위 폴더까지 포함하지 않기 때문에 유저의 접근을 허용하지 않는 의도가 있다. 

예시 => `auth/login/_components/input`
