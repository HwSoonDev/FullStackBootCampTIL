
## 1. 구글 폰트를 사용하자

### 구글 폰트 적용 방법

**왜 구글 폰트를 쓸까?**
- 무료
- Next.js에서 바로 쓸 수 있게 도와주는 도구(next/font/google)도 제공하니까
- css에서 **@import**로 불러오는 것 보다 성능도 좋고, **자동최적화** 가능

전통적인 import 방식
```css
@import url('<https://fonts.googleapis.com/css2family=Inter:wght@400;700&display=swap>');
```
- 폰트가 외부(Google Fonts 서버)에서 로드됨
- 폰트 로딩 타이밍, 캐싱, 중복 요청 등에 대한 **최적화가 없음**
- 사용자 경험이 나쁠 수 있음 (폰트가 늦게 로딩되면 FOUT, FOIT 현상 발생)

**FOUT, FOIT**
- FOUT(Flash of Unstyled Text) : 기본 시스템 폰트가 먼저 잠깐 보였다가 나중에 웹폰트로 바뀌는 현상
- FOIT(Flash of Invisible Text) : 텍스트가 아예 보이지 않다가 폰트가 로딩된 뒤에야 한꺼번에 보이는 현상

**next/font/google**

- **Next.js가 빌드 시점에 폰트를 직접 다운로드해서 로컬에서 제공함**
- 사용하지 않는 스타일(굵기, 이탤릭 등)은 **자동 제거해서 번들 사이즈 줄임**
- display: swap, preload, font-display 등 최적 세팅 자동 적용(FOIT. FOUT을 해결하기 위한 방법을 자동 적용해준다는 의미)
- 캐싱과 로딩 순서도 최적화 (Layout Shift 방지)

> 즉, 브라우저가 외부 요청을 줄이고, 퍼포먼스 점수가 올라감 (Lighthouse 점수 좋아짐)

## 2. 그 중에서도 구글 `가변 폰트`를 사용하자 

구글폰트 내에서도 `variable font(가변 폰트)`를 선택하는 것이 가장 권장됨

> Variable Font 란? 하나의 폰트 파일에 여러 스타일 변형이 포함된 현대적인 폰트 기술. 전통적인 폰트가 Light, Regular, Bold 같은 몇 가지 고정된 굵기만 제공하는 반면, 가변 폰트는 100부터 900까지 모든 중간 값(예: 352, 475)을 지원

`<기존 방식>`

- 폰트를 바꾸려면 Light, Regular, Bold 등 **각 굵기마다 따로 폰트 파일**을 불러와야 했다
- Light: Inter-Light.woff2
- Bold: Inter-Bold.woff2

`<Vriable Font를 이용한 방식>`

- 하나의 폰트 파일에 모든 굵기가 다 들어있다.
- 굵기 100 ~ 900 사이의 어떤 값도 지원 가능 (예: 354 같은 미세한 굵기도 가능!)
- 웹 성능에도 좋고 스타일 유연성도 커짐

className={inter.className} 적용 방식

```jsx
import { Inter } from 'next/font/google'
 
// If loading a variable font, you don't need to specify the font weight
const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
})

// inter.className은 'font-inter'같은 클래스를 의미
export default function RootLayout({ children }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

className={inter.variable} 적용 방식
```jsx
import { Inter } from 'next/font/google'
 
// If loading a variable font, you don't need to specify the font weight
const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-inter' // CSS 변수 이름을 지정
})
 
export default function RootLayout({ children }) {
  return (
    <html lang="en" className={inter.variable}>
      <body className="font-[family-name:var(--font-inter)]">
        {children}
      </body>
    </html>
  )
}
```
