프론트가 백엔드가 되는 기술

app/ 폴더 내에 route.js 파일을 생성하면, 해당 경로에 API 엔드포인트가 만들어진다. 이 파일은 서버에서만 실행되며 GET, POST 등의 HTTP 요청을 처리할 수 있다.
=> 쉽게 말해 클라이언트가 서버의 역할을 하게 되는 기술이다.

![[Pasted image 20251229091024.png]]


언제 사용할까?

1. Client Component 에서 api 요청 시 민감한 정보(인증, api key, api url, token 등) 노출 방지 (`정보보안`)
2. Client Component 에서 api 요청 시 `CORS 우회`
3. `외부 클라이언트에 REST API 제공`하고 싶을 때 >> 이론상 완전가능! 백엔드가 되어버려

### GET 예시

```js
export async function GET() {
  const res = await fetch('https://data.mongodb-api.com/...', {
    headers: {
      'Content-Type': 'application/json',
      'API-Key': process.env.DATA_API_KEY,
    },
  })
  const data = await res.json()
 
  return Response.json({ data })
}
```

Dynamic Routes

```js
// app/items/[id]/route.js
export async function GET(
  request,
  { params }
) {
  const { id } = await params // 'a', 'b', or 'c'
}
```

쿼리

```js
import { type NextRequest } from 'next/server'
 
export function GET(request: NextRequest) {
  const searchParams = request.nextUrl.searchParams
  const query = searchParams.get('query')
  // query is "hello" for /api/search?query=hello
}
```

### POST 예시

```js
export async function POST(request) {
	// request body 추출
  const body = await request.json()
  await fetch(`${API_URL}/todo`, {
	  method: "POST",
		headers: {
			'Content-Type': 'application/json'
		},
		body: JSON.stringfy(body)
  })
  return Response.json({ body })
}
```
