

==`복잡한 낙관적 UI를 사용할때만 부분적으로 ReactQuery를 적용하고,`==
==`그 외에는 Next.js 기반의 서버 액션으로 구현한다`==





---

|상황|추천 방식|
|---|---|
|일반 CRUD, 폼 제출, 리스트/디테일|Next.js 서버 액션 + revalidatePath / revalidateTag (또는 cache: 'no-store')|
|복잡한 낙관적 UI (즉시 반영 후 롤백 등)|React Query (또는 SWR)를 그 부분만 사용|

### React Query를 “부분적으로” 쓰는 이유

- 서버 액션 + revalidate:

서버 처리 → 무효화 → 다시 fetch → 렌더.

사용자는 “제출 후 잠깐 기다리면 화면이 갱신되는” 흐름.

- 낙관적 UI:

제출 직후 클라이언트에서 먼저 UI를 바꾸고, 서버 응답에 따라 성공 유지 또는 롤백.

이때 “클라이언트 캐시를 즉시 수정/롤백”하는 건 React Query가 잘 맞고, Next.js 캐시만으로는 구현이 번거롭습니다.
