
```JSX
useEffect(() => {
	handleLoad(order);  
}, [order]); // <- 의존성 배열
```
컴포넌트가 재렌더링되어도 다시 실행되지 않는 코드를 넣는 훅.
처음 실행 시 콜백 함수를 기억한 후 실행한다.
다시 재렌더링되어도 콜백함수를 실행하지 않는다.

**"의존성 배열"** 안에 상태(State) 변수를 넣어서, 해당 State의 변화가 감지 될 시에 다시 실행될 수 있다.

주로 사이드 이펙트를 관리 하는데 사용한다.

## 주요 3가지 기능
#### 1. 처음 렌더링 후에 한 번만 실행 (초기화)

```JSX
useEffect(() => {
  // ...
}, []);
```

#### 2. 렌더링 후에 특정 값이 바뀌었으면 실행 (최신화)

```JSX
useEffect(() => {
  // ...
}, [dep1, dep2, dep3, ...]);
```

#### 3. 사이드 이펙트 정리(Cleanup)하기 (사이드 이펙트 정리)

```JSX
useEffect(() => {
  // 사이드 이펙트

  return () => {
    // 정리
  }
}, [dep1, dep2, dep3, ...]);
```

사이드 이펙트: 외부 변수에 영향을 주는 것.

useEffect는 
1) 사이드 이펙트를 일으키기 위해 
2) 사이드 이펙트를 일으키지 않기 위해

사용한다.

리액트에서 **State가 바뀌면 해당 컴포넌트를 재렌더링** 하는 특징이 있다.
UI 변경 뿐만 아니라, 컴포넌트 내에서 정의하는 변수, 함수를 새로 다시 정의하기도 하고, 
실행되는 함수가 있다면 다시 실행한다.

하지만, 재렌더링이 일어나도 다시 실행하고 싶지 않은 구문도 존재할 것이다.
이런 구문을 useEffect 안에 넣으면 실행되지 않는다.
외부에 영향을 주지 않는 코드를 구현할 수 있는 것이다.
반대로 조건이 맞는다면, 실행하고 싶은 구문도 있을 것이다.
의존성 배열에 조건에 해당하는 State를 넣는 것 만으로 실행하고 싶은 코드를 제어할 수 있다.

useEffect는 이름 그대로 리액트에서 발생하는 이펙트를 제어하는 역할을 한다.
