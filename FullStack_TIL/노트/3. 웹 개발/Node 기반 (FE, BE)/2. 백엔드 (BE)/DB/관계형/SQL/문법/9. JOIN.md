```
INNER JOIN - 두 테이블의 겹치는 행 (한 컬럼을 기준으로)
LEFT JOIN - 왼쪽 테이블의 행을 모두 출력 (거기에 오른쪽 테이블의 데이터를 추가)
RIGHT JOIN - 오른쪽 테이블의 행을 모두 출력 (LEFT의 반대지만 잘 쓰이지 x)
FULL JOIN - 두 테이블의 모든 행을 출력 (데이터 처리를 할 때 이렇게 마구잡이로 합치는 경우는 없어서 이것도 잘 쓰이지는 x)
```

각각 다른 이름으로도 불리는데, LEFT OUTER JOIN, RIGHT OUTER JOIN, FULL OUTER JOIN 모두 같은 말이다. 알아두면 좋다.

# `INNER JOIN` 

두 테이블의 열을 기준으로 공통으로 있는 튜플을 묶고, 표로 만든다.
1) 테이블1, 2에 공통으로 있는 열만 출력한다.
2) 테이블이 여러 개 사용될 때는 as를 사용해서 편하게 사용하자.

![[Pasted image 20260123154230.png]]

```sql
SELECT 열이름
FROM 테이블1
INNER JOIN 테이블2
ON 테이블1.열이름 = 테이블2.열이름
```

```sql
SELECT 테이블1.열이름 테이블2.열이름 (이런식으로 JOIN 후 필요한 컬럼을 뽑을 수도 있음)
FROM 테이블1
INNER JOIN 테이블2
ON 테이블1.열이름 = 테이블2.열이름
```

# `LEFT JOIN`

![[Pasted image 20260123165820.png]]

```sql
SELECT 열이름
FROM 테이블1
LEFT JOIN 테이블2
ON 테이블1.열이름 = 테이블2.열이름
```

1) LEFT JOIN은 테이블 1(왼쪽)의 모든 행을 출력한다.
(테이블1에 없는 테이블 2의 행은 출력하지 않는다.)

2) 테이블1(왼쪽)에 있는 행 중에 테이블2에 없는 행은 컬럼이 빈 칸이 된다.
3) 테이블1(왼쪽)에 있는 행이라면 중복된 행이라도 모두 출력된다.

# `RIGHT JOIN`

```sql
SELECT 열이름
FROM 테이블1
RIGHT JOIN 테이블2
ON 테이블1.열이름 = 테이블2.열이름
```

반대로 테이블2(오른쪽)의 행을 모두 출력하는 기능이다.

RIGHT JOIN 뒤에 작성한 테이블을 기준으로 행을 표시하는 것인데,

보통 데이터를 가져올 떄 테이블1을 기준으로 2를 붙이는 경우가 많기 떄문에 
사용되지 않는다.
