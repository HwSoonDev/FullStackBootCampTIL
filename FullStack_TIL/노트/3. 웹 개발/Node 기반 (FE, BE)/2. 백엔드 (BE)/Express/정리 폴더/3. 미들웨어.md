## 내장 미들웨어 

엔드 포인트에서 실행하는 함수다.

```js
app.메서드('url엔드포인트', 미들웨어1, 미들웨어2, 미들웨어3...)
```

엔드 포인트의 형태는 위와 같다.

각 라우터(엔드포인트, 주소 끝 단)에 요청이 오면, 미들웨어를 실행하게 되는데,

미들웨어의 종류는 크게 3가지가 있다.

1. 매개변수 2개 (req, res)
2. 매개변수 3개 (req, res, next)
3. 매개변수 4개 (err, req, res, next)

```js
function firstMiddleware(req, res, next) {
  console.log("첫 번째 미들웨어 실행");
  next(new Error("첫 번째 미들웨어에서 에러발생")); // 에러 핸들러로 에러 전달
}

function secondMiddleware(req, res) {
  console.log("두 번째 미들웨어 실행");
  res.json({ message: "응답 대령이오~!" })
}

function errorHandler(err, req, res, next) {
	console.log("에러 핸들러 실행");
  console.log(err.message);
  res.json({ message: err.message })
}
```

2개짜리는 일반적인 미들웨어이고, 
(보통은 가장 마지막에 메인 기능을 수행해서 "라우트 핸들러"라고 부른다.)

3개짜리는 next 인자를 받아서 다음 미들웨어를 실행하게 된다.
미들웨어는 엔드포인트에 인자로 들어간 순서대로 실행되고, 다음 미들웨어 실행을 위해서는 next를 필요로 한다.

4개는 에러 처리다.
미들웨어에서 에러가 발생할 경우 에러처리 미들웨어를 바로 실행한다.
반드시 마지막 인자로 넣어야 한다.

## 인가 처리

미들웨어의 대표적인 활용이 인가 처리다.
인증, 인가여부만 판단하는 미들웨어를 만들어서,
구체적인 기능을 담당하는 미들웨어 앞에 넣기만 해도,
인가된 요청만 걸러내도록 할 수 있다.

예시 2) DB 정보 받아오기 (인증, 인가는 아님)
```js
import express from 'express';

const app = express();

const products = {
  1: {
    name: 'MacBook Pro',
    price: 2500000,
    reviews: [
      { id: 1, user: '민준', rating: 5 },
      { id: 2, user: '서연', rating: 4 },
    ],
  },
  2: {
    name: 'Logitech MX Keys',
    price: 120000,
    reviews: [
      { id: 3, user: '지후', rating: 5 },
      { id: 4, user: '하윤', rating: 3 },
    ],
  },
};

function getProduct(req, res, next) {
  // 여기에 코드를 작성해 주세요.
  const product = products[parseInt(req.params.id)];
  if(product){
    req.product = product;
    next();
  }else{
    res.status(401).json({ message: "존재하지 않는 상품입니다." });
  }
}

app.get('/products/:id', getProduct, (req, res, next) => {
  res.json(req.product);
});

app.get('/products/:id/reviews', getProduct, (req, res, next) => {
  res.json(req.product.reviews);
});

app.listen(3000, () => {
  console.log('Server is listening on port 3000');
});
```
