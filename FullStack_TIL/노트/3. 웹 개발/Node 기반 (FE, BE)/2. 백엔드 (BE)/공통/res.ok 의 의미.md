
기본적으로 `res.status()`가 200~299 사이일때,
프론트에서는 `res.ok = true`로 인식을 한다.

하지만, 응답이 성공적으로 되었어도, 
백엔드 입장에서 내부적으로는 실패를 했을 때, 별도로 표현하고 싶을 수도 있다.
(유저는 존재하지만 권한이 없음.. 등)

이때 사용하는 것이 
`{ ok: false }`
인데, 이건 `res.json().ok`이고 보통 `data.ok`로 받아온다.

```js
.then((res) => res.json())
.then((data) => { if(data.ok) {} })
```

`res.ok`는 200 OK 이지만,
`data.ok`는 `false`일 수도 있다.

정리하자면, 
1) 프론트의 res.ok 처리를 위해서는
   백엔드에서 **상태 코드를 잘 작성**해 주어야 한다는 것.
2) 좀 더 유연한 처리를 위해서 `data.ok`도 활용할 수 있다는 것.
