### Key Derivation Function (키 파생 함수)

키 파생 함수는 비밀번호를 안전하게 저장하는 고급 방법입니다. 일반 해시보다 훨씬 더 강력한 보안을 제공합니다.

## 왜 단순 해싱보다 키 파생 함수가 더 좋을까요?

단순 해싱의 문제점:

- 컴퓨터가 점점 빨라져서 해커의 공격 속도도 빨라짐
- 특수 하드웨어(GPU)로 초당 수십억 개의 해시 계산 가능

키 파생 함수의 해결책:

- **의도적으로 느리게** 동작하도록 설계됨
- 메모리를 많이 사용하게 만들어 병렬 공격을 어렵게 함
- 해시를 여러 번 반복하여 계산 비용을 높임

## 키 스트레칭(Key Stretching)이란?

키 파생함수의 핵심기술로, **해시를 여러 번 반복**하는 기법입니다.

예시:

1. 비밀번호 + 솔트 → 해시
2. 해시 결과 → 다시 해시
3. 이 과정을 수천, 수만 번 반복

🔑 **핵심 아이디어**: 정상 사용자는 로그인할 때 한 번만 기다리면 되지만, 해커는 모든 비밀번호 시도마다 오래 기다려야 합니다!

잘 설계된 시스템은 비밀번호 확인에 0.2초 이상 걸리도록 설정합니다. 컴퓨터가 빨라지면 반복 횟수를 늘려서 대응할 수 있습니다.

## 주요 키 파생 함수 비교

### 1. PBKDF2 (1990년대)

```jsx
// 사용 예시
const hash = crypto.pbkdf2Sync(
  '비밀번호',
  '솔트',
  10000,// 반복 횟수
  64,// 출력 길이
  'sha256'
).toString('hex');
```

- **장점**: 미국 정부 표준으로 승인됨, 간단하고 검증됨
- **특징**: 반복 횟수를 쉽게 조절할 수 있음
- **한계**: 메모리 사용량이 적어 고성능 하드웨어에 취약할 수 있음

### 2. bcrypt (1999년)

```jsx
// 사용 예시
const salt = await bcrypt.genSalt(10);// 2^10번 반복
const hash = await bcrypt.hash('비밀번호', salt);
```

- **장점**: 오랜 시간 검증됨, 사용하기 쉬움
- **특징**: `saltRounds`가 10이면 2^10(=1,024)번 작업 수행
- **제한**: 최대 72바이트 입력만 처리 가능 (긴 비밀번호는 잘림)

### 3. scrypt (2012년)

```jsx
// 사용 예시
const hash = crypto.scryptSync(
  '비밀번호',
  '솔트',
  64,// 출력 길이
  { N: 1024 }// 계산 비용 설정
);
```

- **장점**: 메모리 사용량이 많아 특수 하드웨어 공격에 강함
- **특징**: PBKDF2보다 안전하고 bcrypt보다 경쟁력 있음
- **원리**: 많은 메모리를 사용해서 병렬 공격을 어렵게 만듦

### 4. Argon2 (최신)

```jsx
// 사용 예시
const hash = await argon2.hash('비밀번호', {
  timeCost: 3,// 시간 비용
  memoryCost: 4096,// 메모리 사용량
  parallelism: 1// 병렬 처리 수준
});
```

- **장점**: 가장 현대적이고 강력한 알고리즘
- **특징**: 시간, 메모리, 병렬화 수준을 모두 조절 가능
- **추천**: 새 프로젝트라면 이 알고리즘 선택 고려
