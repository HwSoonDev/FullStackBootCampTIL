
세션 방식


세션이란, 서버가 사용자마다 열어주는 기록 공간이다.
세션 미들웨어를 사용하고, req.session.변수로 변수를 저장해주면
리퀘스트를 보내오는 사용자(PC)별로 세션을 열어 저장해준다.

한번 로그인에 성공해 세션에 기록되면, 
다시 로그인을 하지 않아도 인가된 행동을 할 수 있게 된다.

----------
서버가 세션에 유저 ID를 기록할 때,
동시에 사용자 브라우저 쿠키에도 유저 ID를 기록한다.

로그인 후 다른 요청을 보낼 시 이 쿠키가 자등으로 리퀘스트에 포함되어 함께 보내진다.
(즉, 프론트에서 처리할 내용은 없는 것.)
req.session에서 userId를 읽어들일 수 있는 이유이다.

### 1. 세션 미들웨어
```js
import session from 'express-session';

	app.use(
	  session({
	    secret: process.env.SESSION_SECRET,
	    resave: false, // 변경사항 없을 때는 세션을 다시 저장하지 않음
	    saveUninitialized: true, // 게스트도 포함하여 세션관리 할 때
	  }),
	);
```

=> app.use를 사용하여 서버에서 세션을 사용하도록 미들웨어를 추가한다.

### 2. 로그인 엔드포인트
```js
userController.post("/session-login", async (req, res, next) => {
  const { email, password } = req.body;
  try {
    if (!email || !password) {
      const error = new Error("email, password 가 모두 필요합니다.");
      error.code = 422;
      throw error;
    }
    const user = await userService.getUser(email, password);
    req.session.userId = user.id;
    res.json(user);
  } catch (error) {
    next(error);
  }
});
```

```
req.session.userId = user.id;
```

로그인 엔드포인트에 세션에 유저 아이디를 저장하는 문장을 추가해주면,
로그인 성공 시 유저 정보를 body 담아 리스폰스로 돌려보내기도 하지만, 
서버 자체적으로 세션에 유저 아이디를 저장해서 관리하게 된다.

### 3. 인가 정보 검사 미들웨어

로그인에 성공 후에는 인가가 필요한 요청을 보낼때,
인가되었는지를 세션을 보고 검사하는 미들웨어를 만들어 준다.

```
throwUnauthorizedError();
```
이 함수는 비인가된 경우 오류를 던지는 부분을 반복하기 귀찮아서 묶어놓은 단순한 함수다.

```
const { userId } = req.session;

if (!userId) {
  // 로그인되어있지 않으면 인증 실패
  throwUnauthorizedError();
}
```
리퀘스트에 전달된, 세션 정보에서 유저 아이디가 있는지 확인한다.
있다면, DB에 일치하는 유저가 있는지도 확인한다.

```js
import userRepository from '../repositories/userRepository.js';

function throwUnauthorizedError() {
  // 인증되지 않은 경우 401 에러를 발생시키는 함수
  const error = new Error('Unauthorized');
  error.code = 401;
  throw error;
}

async function verifySessionLogin(req, res, next) {
  // 세션에서 사용자 정보를 읽어옴
  try {
    const { userId } = req.session;

    if (!userId) {
      // 로그인되어있지 않으면 인증 실패
      throwUnauthorizedError();
    }

    const user = await userRepository.findById(req.session.userId);

    if (!user) {
      throwUnauthorizedError();
    }

    // 이후 편리성을 위한 유저 정보 전달
    req.user = {
      id: req.session.userId,
      email: user.email,
      name: user.name,
      provider: user.provider,
      providerId: user.providerId,
    };
    // 사용자가 로그인되어 있다면 다음 미들웨어 처리
    next();
  } catch (error) {
    next(error);
  }
}

export default {
  verifySessionLogin,
}
```
