📦 package.json의 개념

- **프로젝트 메타데이터 파일**
	- 프로젝트 이름, 버전, 설명, 라이선스 등 기본 정보 포함

- **의존성 관리**
	- 어떤 라이브러리(패키지)를 사용하는지, 버전은 무엇인지 기록

- **스크립트 정의**
	- `npm run start`, `npm run build` 같은 실행 명령어를 정의 가능

- **환경 설정**
	- 엔진(Node.js 버전), 설정 옵션 등을 명시

🔗 Dependency 관리의 개념

- **Dependencies**: 실제 실행 시 필요한 라이브러리 (예: Express, React)
- **DevDependencies**: 개발 과정에서만 필요한 라이브러리 (예: Jest, ESLint, Webpack)
- **PeerDependencies**: 특정 환경에서 함께 설치되어야 하는 라이브러리 (예: React plugin이React 버전에 의존)
- **OptionalDependencies**: 없어도 실행 가능한 선택적 라이브러리

⚡ 중요성

1. **재현성 (Reproducibility)**
	- `package.json`과 `package-lock.json`을 통해 다른 개발자가 동일한 환경을 쉽게 재현 가능.

2. **협업 효율성**
	- 팀원들이 같은 버전의 라이브러리를 사용 → 버전 충돌 방지.

3. **보안 관리**
	- 의존성 업데이트를 통해 취약점 패치 가능.
	- 오래된 라이브러리 방치 시 보안 위험 증가.

4. **프로젝트 확장성**
	- 라이브러리 버전 관리가 잘 되어야 프로젝트가 안정적으로 성장 가능.

5. **자동화와 배포**
	- 빌드, 테스트, 배포 과정에서 필요한 도구들을 명확히 정의해 자동화 파이프라인 구축에 필수.

💡 정리
- `package.json`은 **프로젝트의 청사진**이고,
- Dependency 관리는 **프로젝트의 건강 상태**를 유지하는 핵심입니다.
