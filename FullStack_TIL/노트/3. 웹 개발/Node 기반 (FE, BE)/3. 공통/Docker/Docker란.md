컨테이너 기반 오픈소스 가상화 플랫폼입니다.
환경 전체를 담아서 옮기고, 관리할 수 있는 "컨테이너"라는 개념을 사용합니다.
가상화라고 하면 Virtual Machine처럼 물리적으로 다른 하드웨어를 사용할 것 같지만, 
Docker의 컨테이너는 이미지(image)로 환경과 파일을 함께 관리하는 형태로, 기본은 리눅스 기반이기 때문에 무조건 가상 머신인 것은 아닙니다!

## 컨테이너란?

![[Pasted image 20260329114325.png]]

OS 상에 논리적인 영역을 구축하고, 애플리케이션이 동작하는데 필요한 요소들을 모아 별도의 서버처럼 동작하는 것입니다. 필요한 요소만으로 구성되어 있어서 오버헤드가 적습니다.

## 왜 가상화 플랫폼인가?

![[Pasted image 20260329114917.png]]

Docker는 리눅스 기반의 컨테이너를 사용하기 때문에, 사용하는 OS가 리눅스가 아니라면 가상 머신 위에서 동작합니다. (이 경우 무겁기는 합니다.) 그럼에도 얻는 이득이 많기 때문에 리눅스 가상머신을 사용합니다!

## 이미지 (image)

도커 컨테이너에서 이미지는 정말 중요한 개념입니다.

이미지란, "이미 만들어진 환경"을 말합니다.

예를 들어서, DB를 사용하고 싶다면,
로컬의 경우:
1) DB 프로그램을 로컬 PC에 설치 (MySQL, postgeSQL)
2) 로컬 백엔드와 연결
의 순서로 진행합니다.

하지만, 
도커의 경우:
1) DB를 설치할 컨테이너를 만듦 
2) DB가 설치된 이미지(이미 만들어진 환경)를 다운로드
3) Docker Hub에서 다운로드 진행
4) 백엔드가 설치된 컨테이너와 연결
의 순서로 진행됩니다.

어떤 환경을 처음부터 설치하는 것이 아니라, 이미 준비된 환경(image)을 다운로드 받는다.
이것이 도커의 핵심입니다.

```docker-compose.yml
services:
  db:
    image: postgres:16-alpine //<- 이게 이미지
```
