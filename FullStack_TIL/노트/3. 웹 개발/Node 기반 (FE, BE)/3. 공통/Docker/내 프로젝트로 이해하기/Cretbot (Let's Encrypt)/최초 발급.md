
```bash
docker compose run --rm certbot certonly \
  --webroot -w /var/www/certbot \
  -d example.com \ # 사용할 도메인 주소 (등록된 걸로)
  --email you@example.com \  # let's ecrypt에서 연락을 받을 나의 이메일
  --agree-tos --no-eff-email
```
 
 https 인증서를 최초 발급할 때는 
 1) 도커의 nginx가 실행되고 있는 상태로,
 2) 위와 같은 명령어를 입력합니다.

+번외)
저는 entrypoint 설정을 compose 파일에 작성해서 덮어씌워야 한다고 합니다.
그래서 이렇게 `--entrypoint certbot`를 추가해주었습니다. (원래는 필요 없음)

```bash
sudo docker compose run --rm --entrypoint certbot certbot certonly 
--webroot -w /var/www/certbot 
-d <도메인 주소>
--email <이메일>
--agree-tos --no-eff-email
```


백그라운드 실행
```bash
sudo docker compose up -d #전체 컨테이너 백그라운드 실행
sudo docker compose up -d nginx #nginx만 실행
```
