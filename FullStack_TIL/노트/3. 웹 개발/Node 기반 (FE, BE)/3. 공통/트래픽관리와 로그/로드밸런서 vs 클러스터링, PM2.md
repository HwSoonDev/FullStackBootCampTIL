## 로드 밸런서(Load Balancer) 
-> 여러 대의 서버 활용

로드 밸런서는 **들어오는 네트워크 트래픽을 여러 서버에 분산시켜주는 장치(혹은 소프트웨어)**예요.  
쉽게 말해, **많은 사람들이 동시에 웹사이트에 접속할 때 한 서버가 과부하되지 않도록 트래픽을 나눠주는 역할**을 합니다.

### ⚙️ 주요 기능

- **트래픽 분산**: 요청을 여러 서버에 고르게 배분해 성능과 안정성을 높임
- **장애 대응**: 특정 서버가 다운되면 자동으로 다른 서버로 요청을 보내 서비스 중단을 최소화
- **확장성 확보**: 서버를 추가하면 로드 밸런서가 알아서 트래픽을 새 서버에도 분배
- **보안 강화**: 외부에서 직접 서버에 접근하지 못하게 하고, 로드 밸런서가 앞단에서 보호막 역할 수행

### 🖥️ 예시 상황

- **웹사이트 접속**: 사용자가 동시에 1만 명 접속 → 로드 밸런서가 요청을 10대 서버에 나눠서 처리
- **서버 장애**: 1대 서버가 다운 → 로드 밸런서가 자동으로 나머지 9대 서버에만 요청 전달
- **확장**: 이벤트 시즌에 서버를 5대 더 추가 → 로드 밸런서가 새 서버에도 트래픽을 분배

### 📊 로드 밸런서 종류

|종류|특징|
|---|---|
|**하드웨어 로드 밸런서**|전용 장비, 고성능, 비용이 높음|
|**소프트웨어 로드 밸런서**|Nginx, HAProxy 같은 오픈소스, 클라우드 환경에서 많이 사용|
|**클라우드 로드 밸런서**|AWS ELB, Azure Load Balancer, GCP Load Balancing 등 클라우드 제공 서비스|

정리하면, 로드 밸런서는 **웹사이트나 서비스가 많은 트래픽을 안정적으로 처리할 수 있도록 트래픽을 분산시키는 핵심 인프라**예요.

혹시 원하시면, 제가 **로드 밸런서가 동작하는 구조를 그림으로 시각화**해 드릴 수도 있어요. 원해요?

## 클러스터링 
-> 하나의 서버에서 멀티코어 활용
###  Node.js 멀티코어 활용 (클러스터링 / PM2 등)

- Node.js는 기본적으로 **싱글 스레드, 싱글 코어**에서 동작합니다.
- 하지만 운영체제는 멀티코어 CPU를 가지고 있으니, 이를 활용하기 위해 **클러스터 모듈**이나 **PM2 같은 프로세스 매니저**를 사용합니다.
- 동작 방식:
    - 하나의 서버 머신 안에서 **여러 Node.js 프로세스**를 띄움
    - 운영체제의 IPC(Inter-Process Communication)나 내부 라운드 로빈 방식으로 요청을 각 프로세스에 분배
- 즉, **한 서버(한 머신) 안에서 CPU 코어를 효율적으로 쓰기 위한 분배**입니다.

### 비교 정리

|구분|Node.js 클러스터/PM2|로드 밸런서|
|---|---|---|
|범위|한 서버 내부 (멀티코어 활용)|여러 서버 간 (멀티머신 활용)|
|목적|CPU 코어 활용 극대화|트래픽 분산, 장애 대응, 확장성|
|구현|Node.js 클러스터 모듈, PM2|Nginx, HAProxy, AWS ELB 등|
|장애 대응|프로세스 단위 재시작|서버 단위 트래픽 우회|

## 클러스터링, PM2의 관계

 **클러스터링 (Clustering)**

- **개념**: 
	- 하나의 서버에서 여러 프로세스를 띄워 CPU 멀티코어를 활용하는 방식
	- Node.js는 기본적으로 싱글 스레드라서, 클러스터링을 통해 여러 프로세스를 병렬로 실행해 성능을 높임
- **구현 방법**:
	- Node.js의 **내장 `cluster` 모듈** 사용
	- 직접 코드에서 `cluster.fork()`로 워커 프로세스를 여러 개 생성

-> 즉, **멀티코어 활용을 위한 기술적 개념**이에요.

**PM2**

- **도구(프로세스 매니저)**: 
	- Node.js 애플리케이션을 관리하는 데 특화된 툴
- **주요 기능**:
	- 클러스터 모드 지원 (내부적으로 Node.js `cluster` 모듈을 사용)
	- 프로세스 자동 재시작 (장애 발생 시)
	- 로드 밸런싱 (멀티 프로세스 간 요청 분배)
	- 로그 관리, 모니터링, 배포 관리 등 운영 편의 기능 제공

-> 즉, **클러스터링을 쉽게 적용하고 운영 관리까지 해주는 툴**이에요.

✅ 결론:

- **클러스터링은 멀티코어 활용을 위한 개념/기술**이고,
- **PM2는 그 클러스터링을 쉽게 적용하고 운영까지 관리해주는 도구**입니다.

## 클러스터링을 로드밸런서처럼 쓸 수 있을까?


- 한 컴퓨터에는 여러 CPU 코어가 있습니다.
- Node.js 같은 싱글 스레드 서버는 기본적으로 한 코어만 사용하지만, **클러스터링**이나 **PM2** 같은 프로세스 매니저를 쓰면 여러 프로세스를 띄워 각 코어에 배치할 수 있습니다.
- 운영체제(OS)가 요청을 여러 프로세스에 분산시켜서 **트래픽을 나누어 처리**하게 됩니다.
- 즉, **한 컴퓨터 안에서 여러 개의 서버 프로세스를 띄우고 트래픽을 분산하는 것**이 가능합니다.

🌐 로드밸런서와의 차이

- **멀티코어 클러스터링**: 한 컴퓨터 내부에서 CPU 코어를 활용해 여러 프로세스를 운영 → 트래픽을 프로세스 단위로 분산
- **로드밸런서**: 여러 대의 컴퓨터(서버) 사이에서 트래픽을 분산 → 서버 단위로 분산

➡️ 한 컴퓨터에서 멀티코어를 활용하면 "작은 규모의 로드밸런싱"처럼 보일 수 있지만, **실제 로드밸런서처럼 서버 장애 대응이나 확장성 확보는 어렵습니다.**
