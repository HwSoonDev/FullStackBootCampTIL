
## 네트워크 비유 
- 논리적 LAN = 어쩌구 마을  → AWS VPC, 한 덩어리의 네트워크 전체 
- 서브넷 = 저쩌구동, 절씨구동 → 목적/보안/라우팅에 따라 나눈 네트워크 블록 
- Public Subnet → 외부에서 접근 가능한 구역 
- Private Subnet → 외부에서 접근 불가, 내부용 
- IP 주소 = 홍길동 집 → 그 안의 서버 한 대, EC2나 RDS 각각이 집 하나를 갖고 있음 → 집 주소(IP)가 있어야 마을 안에서 통신 가능 
- MAC 주소 = 집주인 고유 번호 → 홍길동 이름, 변하지 않음, 네트워크 내부에서 장치 식별 

## AWS가 서브넷을 요구하는 이유 
- 단순 IP 수 관리 (X) - `보안, 라우팅, 접근 제어`를 위해 네트워크를 나누는 것이 필수 
- Public/Private 구분 없으면 DB가 인터넷에 노출될 수 있고, 서버 관리도 힘듦 ⇒ 한 마을 안에서도 건물별로 목적과 규칙에 맞춰 구역을 나누는 것이 서브넷의 역할
