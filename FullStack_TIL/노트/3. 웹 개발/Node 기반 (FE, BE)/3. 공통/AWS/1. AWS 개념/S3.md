AWS에서 제공하는 파일 업로드/ 다운로드 기능 DB

1. FE에서 IAM 정보를 가지고 바로 S3에 접근하는 방식 -> pre-signed-URL 필요
2. BE를 거쳐서 S3에 업로드/다운로드 하고 BE에서 FE로 파일을 넘기는 방식. -> 느리기는 해도 IAM 정보, 액세스 키 등을 FE 코드에 노출시키지 않아서 안전한 방식이다. 

### 미리 서명된 URL (업로드/다운로드용 URL)
<중요>
pre-signedURL => 미리 서명된 URL 

업로드/다운로드 할 때 일정 시간만 유효한 URL을 BE에서 받아 S3에 접속을 한다.

### 만료 시간 책정
보통 업로드는 원 액션 방식이라 시간을 짧게 주고,
웹페이지에 상시 표시되는 이미지 등의 경우에는 다운로드 시간을 1시간 정도로 널널하게 준다고 한다. 하지만 만료되면 안보이게 되는 건 똑같은 듯 하다.
