IaaS: 컴퓨터만 제공하는 서버 제공 방식(운영체제 정도만 선택)
PaaS: 개발 환경을 함께 제공하는 서버 제공 방식 (Vercel, Render)
SaaS: 완성된 소프트웨어를 제공하는 방식
[[2) 클라우드 서비스 모델 IaaS, PaaS, SaaS]]

EC2는 이 중에서 IaaS 모델에 해당한다.

### EC2 (Elastic Compute Cloud)

- **클라우드 상의 가상 컴퓨터**
- 언제든지 생성하고 삭제할 수 있는 서버
- 사용한 만큼만 비용 지불
    - 필요한 스펙의 컴퓨터를 필요한 만큼 대여 가능

<EC2 는 가장 범용적인 용도로 사용할 수 있는 서비스입니다.>

---

## 2. EC2의 핵심 개념

### 인스턴스 (Instance)

- **하나의 가상 서버**
- 물리적 컴퓨터를 소프트웨어로 구현
- 독립적인 운영체제와 리소스

### AMI (Amazon Machine Image)

```
AMI = 컴퓨터의 초기 설정
- OS (Windows, Linux, Ubuntu 등)
- 설치된 소프트웨어
- 기본 설정값
```

### 인스턴스 타입

<aside> ☝

4 vCPU는 2 vCPU보다 동시에 더 많은 일을 할 수 있어서 성능이 2배 가까이 좋습니다.

**CPU 연산량이 크거나 트래픽 많으면 4 vCPU로 가는 게 안전해요.**

</aside>
## 3. EC2 생성 과정

### Step 1: AMI (OS) 선택

<aside> ☝

개인 PC(데스크톱)는 Windows가 압도적으로 많지만, 서버 세계(클라우드/백엔드 등)는 Linux가 압도적으로 많아요!

</aside>

```
📀 운영체제 선택
- Amazon Linux 2023 (AWS 의 권장)
- Ubuntu (가장 대중적 서버 OS)
- Windows Server
- CentOS
```

### Step 2: 인스턴스 타입 선택

```
💻 컴퓨터 사양 선택
- CPU 코어 수
- 메모리 크기
- 네트워크 성능
```

### Step 3: 네트워크 설정

```
🌐 네트워크 환경 구성
- VPC (가상 네트워크)
- 서브넷 (네트워크 구간)
- 퍼블릭 IP 할당 여부
```

### Step 4: 보안 그룹 설정

```
🔒 방화벽 규칙 설정
- SSH (22번 포트) - 서버 접속용
- HTTP (80번 포트) - 웹사이트용
- HTTPS (443번 포트) - 보안 웹사이트용
```

### Step 5: 키 페어 생성

```
🔑 서버 접속용 열쇠
- 비밀번호 대신 사용
- .pem 파일로 다운로드
- 분실 시 서버 접속 불가
```

## 4. EC2 접속 방법

### Linux/Mac에서 접속

```bash
# 키 파일 권한 설정
chmod 400 my-key.pem

# SSH 접속
ssh -i my-key.pem ec2-user@퍼블릭IP주소
```

### Windows에서 접속

```
방법 1: AWS 콘솔에서 직접 접속
- EC2 → 인스턴스 선택 → "연결" 버튼

방법 2: PuTTY 사용
- .pem을 .ppk로 변환
- PuTTY로 접속
```

## 5. EC2 요금 구조

### 온디맨드 (On-Demand)

```
💰 시간당 고정 요금
- 언제든 시작/중지 가능
- 약정 없음
- 가장 비싸지만 유연함
```

### 예약 인스턴스 (Reserved Instance)

```
💰 1~3년 약정으로 할인
- 최대 75% 할인
- 장기간 사용 시 유리
```

### 스팟 인스턴스 (Spot Instance)

```
💰 경매 방식으로 최대 90% 할인
- AWS 여유 용량 활용
- 언제든 중단될 수 있음
- 실험용이나 배치 작업에 적합
```

## 6. 보안 베스트 프랙티스

### 보안 그룹 설정

```
✅ 필요한 포트만 열기
✅ 신뢰할 수 있는 IP만 허용
❌ 0.0.0.0/0 (모든 IP) 허용 금지
```

```
✅ 키 파일을 안전한 곳에 보관
✅ 키 파일 권한을 400으로 설정
❌ 키 파일을 공개 저장소에 업로드 금지
```
