**아마존은 왜 aws를 만들게 되었을까요?**
    - 아마존은 쇼핑몰
    - 내부 개발 문제가 너무 커짐
        2000년대 초반, 아마존은 **거대한 온라인 쇼핑몰**을 운영하고 있었는데 내부 개발 환경이 아주 엉망이었어요.
        
        - 팀마다 서버를 따로 운영
        - 인프라가 표준화되어 있지 않음
        - 새로운 기능 하나 배포하려면 몇 주씩 걸림
        - 개발자들은 **비즈니스 로직보다 인프라 세팅에 시간을 더 씀**
        
        그래서 내부적으로:
        
        > “우리가 회사 전체 인프라를 아예 플랫폼으로 만들어버리면 어떨까?”
        
        이런 논의가 나오기 시작함.
        
        결론: **내부 개발 효율성을 위해 공통 인프라 플랫폼을 만들자 → 이것이 AWS의 시작.**
        
    - 남는 컴퓨팅 자원이 너무 많음!
        
        아마존은 블랙프라이데이 같은 이벤트 대비해서 **최대 트래픽 기준으로 서버를 잔뜩 준비**해놨는데…
        
        - 평상시에는 그 서버들이 대부분 **놀고 있는 상태**였음.
        
        그래서:
        
        > “어차피 놀고 있는 서버, 다른 회사들한테 빌려주면 돈 되는 거 아니야?”
        
        라는 아주 단순하고 천재적인 생각이 나옴.
        
        그래서 나온 첫 서비스가 바로
        
        - **S3(스토리지)**
        - **EC2(서버 대여)**
    - 스타트업들이 서버 구축이 너무 힘듦
        
        2000년대 초반 스타트업들은 이런 문제를 겪음:
        
        - 서버 한 대 사서 IDC에 넣어야 함
        - 직접 네트워크 구성해야 함
        - 장애 대비하려면 예비 서버까지 사야 함
        - 비용 1,000~5,000만원 필요
        - 준비 기간 최소 1~2개월
        
        그러니까 서비스 하나 런칭하는 데 인프라가 너무 비싸고 오래 걸리는 거야.
        
        아마존은 여기서 기회를 느낌:
        
        > “우리가 서버를 빌려주고, 필요한 만큼만 쓰고 돈 내면 되게 만들면?”
        
        > → 바로
        > 
        > **클라우드 컴퓨팅**
