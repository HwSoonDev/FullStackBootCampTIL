```
- number: 숫자
- string: 문자열
- boolean: 참/거짓
- ~[]: 배열
- undefined : 정의되지 않음 
- null : 없음
- any : 말그대로 아무 타입이나 가능한 경우. 타입 스크립트를 사용한 이상 any타입은 쓰지 말자. 
- unknown : 런타임 시점 타입 검사가 필요한 타입 (api 리스폰스 등) 어떤 타입으로 받을 지 알 수 없지만, 아무 타입이나 받는 것은 아닌 경우. (중요!)
```

< string | null > 처럼 중복 타입이 가능하고, 실제로 null, undefined는 이렇게 쓰인다.

### unknown 타입

정의 시점에는 리턴을 unknown으로 설정해놓으면,
사용 시 반드시 타입 좁히기를 해야 한다.
이게 unknown 타입의 장점.
