🔢 합타입 (Sum Types / Union Types) - "또는"의 개념_ -> " | "

- 가능한 값들의 합집합

```tsx
type Status = 'loading' | 'success' | 'error';

let userStatus: Status = 'loading';
```

🔢 곱타입 (Product Types) - "그리고"의 개념_ -> 객체, 튜플

- 여러 타입을 동시에 만족함
- 객체 타입 또는 튜플 타입으로 표현

### 1. 객체 타입 (기본적인 Product Type)

```tsx
interface User {
  name: string;// 필수
  age: number;// 필수
  email: string;// 필수
}
```

### 2. 인터섹션 타입 (명시적 Product Type)

```tsx
type PersonalInfo = { name: string; age: number };
type ContactInfo = { email: string; phone: string };
type User = PersonalInfo & ContactInfo;

// User가 되려면 4개 속성 모두 필요
const user: User = {
  name: "김개발",
  age: 25,
  email: "kim@example.com",
  phone: "010-1234-5678"
};
```

### 3. 튜플 타입 (순서가 있는 Product Type)

```tsx
type Coordinates = [number, number, number];
// 첫 번째 number AND 두 번째 number AND 세 번째 number
```
