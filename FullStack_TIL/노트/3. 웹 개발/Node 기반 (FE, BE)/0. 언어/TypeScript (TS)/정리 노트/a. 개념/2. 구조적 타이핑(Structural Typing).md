typeScript는 **이름이 아닌 구조**로 타입을 판단합니다. 
오리처럼 생기고 오리처럼 행동하면 오리로 취급합니다! 🦆

```tsx
interface Animal {
  name: string;
}

interface Dog {
  name: string;
  breed: string;
}

let dog: Dog = { name: 'Buddy', breed: 'Golden' };
let animal: Animal = dog; // ✅ Dog가 Animal 구조를 포함하므로 호환!

// 구조가 맞으면 어떤 객체든 OK
let cat: { name: string; age: number } = { name: 'Fluffy', age: 3 };
let animal2: Animal = cat; // ✅ name이 있어서 Animal로 사용 가능

// 구조가 부족하면 에러
let robot = { id: 123, power: 'on' };
let animal3: Animal = robot; // ❌ name 속성이 없어서 에러!
```

구조적 타이핑은 덕 타이핑과 유사한 의미를 가지지만, 아래의 차이가 있습니다.

- **Duck Typing**: 런타임에 "실제로 꽥꽥거리나?" 확인
    - 즉, 런타임(실행 중)에 실제로 그 행동을 할 수 있느냐를 봄
    - js 같은 동적 언어에서 자주 쓰이는 개념
- **Structural Typing**: 컴파일타임에 "꽥꽥거릴 수 있는 구조인가?" 확인
    - 컴파일 시점(타입 체크 단계)에 구조만 맞으면 호환 가능
    - ts가 채택한 방식
