### 1. 동적 타이핑 언어

TypeScript는 자바스크립트의 상위 집합(**`superset`**)으로, 자바스크립트 문법에 타입 체크를 위한 문법들을 확장한 언어

> 동적 타이핑 언어 vs. 정적 타이핑 언어

- 동적 타입 언어는 런타임(코드실행)에 타입이 결정됩니다. (JS 는 동적 타입 언어)
- 정적 타입 언어는 런타임 이전에 타입이 결정됩니다.
![[Pasted image 20251117101025.png]]
💡동적 타이핑 언어는 타입 실수하기 쉽고, 그 실수를 찾아내기도 어렵다!

1. **`보다 신뢰할 수 있는 코드`**: 타입 체크를 통해 런타임 오류를 줄이고, 코드의 예측 가능성을 높입니다.
2. **`규모 확장성`**: 코드베이스가 커질수록 타입 시스템 덕분에 안전하고 효율적으로 관리가 가능합니다. (faster safely)
3. **`개발 도구 지원`**: 강력한 코드 자동 완성을 지원하여 생산성을 높입니다.

- 타입스크립트 필요성 예시
```tsx
function calculateTotal(price, tax) {
  return price + tax;
}

// 사용할 때 실수
const result1 = calculateTotal("100", "10");  // "10010" (문자열 연결!)
const result2 = calculateTotal(100);          // NaN (undefined + 100)
const result3 = calculateTotal(100, "10%");   // "10010%" (또 문자열!)

console.log(result1); // "10010" - 예상과 다름!
console.log(result2); // NaN - 오류!
console.log(result3); // "10010%" - 완전히 잘못됨!
```

종합하면,
- 컴파일 단계에서 타입 오류 검출하여 런타임 타입 오류 발생가능성을 줄일 수 있다.
- 코드 가독성 및 유지보수성을 향상시킬 수 있다.

### 2. 동작 원리 (트랜스 파일)

 결국 타입스크립트는 JS 로 변환이 되어야만 실행할 수 있습니다. 우리가 주요 개발환경으로 채택하고 있는 `브라우저`와 `Node.js`는 타입스크립트가 아닌 **`자바스크립트`**를 실행할 수 있는 런타임이기 때문에 그래요!
![[Pasted image 20251117101137.png]]

TS(타입스크립트 코드) → `TSC`(타입스크립트 컴파일러) → JS(자바크스립트 코드)
TSC (Typescript Compiler) 란? 타입스크립트 파일에서 타입 검사 후 자바스크립트로 컴파일 해주는 역할을 하는 컴파일러

#### 일반적으로 말하는 컴파일러

![[Pasted image 20251117101150.png]]
- 소스코드를 컴퓨터가 실행할 수 있는 기계어로 번역하는 작업
- 한 프로그래밍 언어에서 다른 프로그래밍 언어로 번역하는 작업

#### 트랜스파일(Transpile)

![[Pasted image 20251117101310.png]]
- 소스코드를 Node.js 나 브라우저와 같은 런타임에서 실행할 수 있는 코드로 변경하는 작업

compile vs transpile

- transpile : **같은 수준의 언어끼리 변환**할 때
- compile : **보통 고급 언어 → 저수준 언어**, 즉 기계어로 바꾸는 것
