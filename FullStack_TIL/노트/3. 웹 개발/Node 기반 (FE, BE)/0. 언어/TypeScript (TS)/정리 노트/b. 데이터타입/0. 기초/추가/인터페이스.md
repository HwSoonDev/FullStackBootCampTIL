기본적으로 인터페이스는 클래스 관련 문법이고,
typescript의 type 객체과 함께 사용되기도 한다고 이해했음 (승찬)
## 주요 차이점

### 1. 확장(Extension) 방식

**Interface: extends 키워드**

```tsx
interface Animal {
  name: string;
}

interface Dog extends Animal {
  breed: string;
}
```

**Type Alias: 교집합(&) 연산자**

```tsx
type Animal = {
  name: string;
};

type Dog = Animal & {
  breed: string;
};
```

### 2. 선언 병합(Declaration Merging)

이름이 같은 인터페이스를 여러번 선언하면 속성이 자동으로 합쳐짐

**Interface: 가능 ✅**

```tsx
interface User {
  id: number;
}

interface User {// 같은 이름으로 다시 선언 가능
  name: string;
}

// 결과: User는 id와 name을 모두 가져 아래와 같은 최종 구조를 갖게됨
interface User {
	id: number;
	name: string
}
```

**Type Alias: 불가능 ❌**

```tsx
type User = {
  id: number;
};

type User = {// ❌ 에러! 중복 선언 불가
  name: string;
};
```

### 3. 유틸리티/매핑 타입

**Interface: 제한적**

```tsx
interface StringKeys = keyof SomeType;// ❌ 불가능
```

**Type Alias: 강력 ✅**

```tsx
// ✅ 다양한 타입 정의 가능
type StringKeys = keyof SomeType;
type UserKeys = 'id' | 'name' | 'email';
type Optional<T> = { [K in keyof T]?: T[K] };
```

## 언제 무엇을 사용할까?

### Interface 사용 권장하는 경우

- **객체 구조 정의**할 때 (팀 컨벤션에 따라 type alias 사용하기도 함)
    
- **라이브러리 API** 정의할 때 (확장성)
    
- 컴포넌트 props 타입 정의 시
    
- **클래스 구현**에 사용할 때
    
    ```tsx
    interface APIResponse {
      status: number;
      data: any;
    }
    
    class UserService implements APIResponse {
    	// 구현
    }
    ```
