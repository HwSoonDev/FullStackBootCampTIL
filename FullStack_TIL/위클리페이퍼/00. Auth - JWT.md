경험을 바탕으로 React 애플리케이션에서 JSON Web Token(JWT)을 사용하여 사용자 인증 시스템을 구현하는 방법에 대해 자세히 설명해주세요. 특히 로그아웃 구현 로직에 대해 설명해주세요

### 내 답변

JWT는 사용자의 ID와 권한 등의 정보와 함께 서버의 서명을 넣어 암호화한 토큰입니다.
JWT를 통해 로그인(사용자 인증 시스템)을 구현하는 과정은 다음과 같습니다.
먼저 유저의 자격 증명(이메일, 비밀번호, oauth 등)을 검증합니다. 
자격이 증명되면 서버에서 JWT를 발급하여, 브라우저에 저장합니다. 이 때 저장소는 3가지 중에선택할 수 있는데, Local storage, Session storage, Cookie 입니다.  
JWT로 만든 액세스 토큰은 XSS 탈취 등의 보안을 우려하여 Cookie가 선호되지만, 로그인 정보을 서버 세션에서 관리하지 않고 브라우저에 분담 시키기 위해서 브라우저의 Local 또는 Session 스토리지를 사용하기도 합니다. (Local은 긴 기간 저장, Session은 브라우저 실행 중에만 저장)
Local/ Session 스토리지에 저장한 경우 인증이 필요한 API 요청의 헤더에 토큰을 넣어줄 필요가 있고, 쿠키에 저장한 경우 자동으로 API 요청에 포함하는 설정을 서버에서 해주기만 하면 됩니다.

로그아웃 구현은 로컬/세션 스토리지의 경우에는 간단합니다. 인증 정보를 브라우저에서 관리하기 떄문에 별도의 서버 요청 없이 클라이언트 사이드에서 JWT를 삭제하면 됩니다. 그러면 인증 정보가 없어서 자동으로  로그아웃 상태가 됩니다. 쿠키는 브라우저에서 직접 지울 수는 있지만, 보통 서버에 요청하여 set-cookie로 만료 처리를 하도록 합니다. 리프레쉬 토큰을 가지고 있다면 함께 만료 처리합니다.
쿠키를 사용하는 경우 HttpOnly 설정만 지킨다면, XSS에는 강하지만, 요청에 자동으로 쿠키를 포함하는 특성 떄문에 CSRF에는 취약합니다. 이 경우 SameSite 설정을 강화하여 다른 사이트에 쿠키가 포함되지 않도록 막을 수 있습니다. 

### (참고) 모범 답안
    
React 애플리케이션에서 JWT를 사용하여 사용자 인증 시스템을 구현하는 방법은 다음과 같습니다. 첫째, 사용자가 로그인할 때 서버는 사용자의 자격 증명을 검증한 후 JWT를 생성하고 이를 사용자에게 반환합니다. 이 토큰은 일반적으로 사용자의 ID와 권한을 암호화하여 포함하며, 서버는 토큰에 서명하여 변조를 방지합니다.

클라이언트(React 앱)에서는 받은 JWT를 localStorage 또는 sessionStorage에 저장하여 관리할 수 있습니다. 저장된 JWT는 서버로 API 요청을 할 때 HTTP 헤더에 포함하여 서버가 요청을 인증할 수 있도록 합니다. 이렇게 하면 사용자가 애플리케이션을 사용하는 동안 JWT를 통해 인증 상태를 유지할 수 있습니다.

로그아웃 구현은 사용자가 로그아웃 요청을 할 때 클라이언트에서 저장된 JWT를 삭제하는 것으로 처리합니다. 일반적으로 localStorage 또는 sessionStorage에서 JWT를 제거하며, 이 작업은 클라이언트 사이드에서 진행됩니다. 이렇게 JWT를 제거하면, 사용자의 세션이 종료되며 이후의 요청은 서버에서 인증되지 않아 접근이 거부됩니다.

로그아웃 시 JWT를 클라이언트에서 삭제하는 것 외에도, 보안을 강화하기 위해 쿠키를 사용하는 경우 HTTPOnly 속성을 설정하여 클라이언트 스크립트가 쿠키에 접근할 수 없도록 할 수 있습니다. 이는 XSS 공격을 통한 토큰 탈취를 방지하는 데 유용합니다. 또한, 서버 측에서는 로그아웃 시 토큰의 유효성을 무효화하는 로직을 구현할 수 있으나, 이는 선택적인 보안 강화 방법입니다.

### 채점 기준

- JWT 기반 인증 시스템 설명 (+3점)
	- JWT의 개념과 작동 원리에 대한 설명이 명확해야 함.
	- JWT를 React 애플리케이션에 구현하는 기본 절차의 설명.
- React에서 JWT를 사용한 인증 구현 (+3점)
	- 클라이언트 사이드에서 JWT를 저장하는 방법(예: localStorage, sessionStorage).
	- 인증 토큰을 사용하여 API 요청 보안을 구현하는 방법.
- 로그아웃 메커니즘 설명 (+4점)
	- 로그아웃 로직의 구현 방법과 이유에 대한 구체적인 설명.
	- 토큰을 클라이언트에서 제거하는 프로세스.
	- 보안을 유지하기 위한 추가 조치들(예: 쿠키의 HTTPOnly 속성 사용).
