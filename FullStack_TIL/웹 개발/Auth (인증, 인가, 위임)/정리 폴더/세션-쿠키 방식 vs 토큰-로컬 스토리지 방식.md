인증, 인가를 구현 할 때,
1) 세션을 받아서 쿠키에 저장하는 방식
2) 토큰을 받아서 locale sotrage에 저장하는 방식
두 가지가 있는데,  각각 장단점이 있다.
#### **리퀘스트 형태**

세션 방식은 브라우저가 자동으로 전송, 관리하고
토큰 방식은 직접 리퀘스트 헤더에 넣어줘야 하는 차이점이 있다.

#### **보안 측면**

XSS 측면에서는 세션이 강하고
CSRF 측면에서는 토큰이 강하다.
### 📊 비교 요약

| 항목           | 세션 쿠키             | 토큰 로컬 스토리지            |
| ------------ | ----------------- | --------------------- |
| 저장 위치        | 쿠키                | 로컬 스토리지               |
| 전송 방식        | 브라우저가 자동 전송       | 직접 헤더에 삽입             |
| 보안 (XSS)     | 비교적 안전 (HttpOnly) | 취약 (JavaScript 접근 가능) |
| 보안 (CSRF)    | 취약 (대응 필요)        | 안전 (자동 전송이 안 되므로)     |
| 서버 부담        | 있음 (세션 저장)        | 없음 (stateless)        |
| REST API 적합성 | 낮음                | 높음                    |
| 토큰 갱신        | 서버에서 자동 관리        | 클라이언트가 직접 처리 필요       |

### 🔐 1. 세션 쿠키 방식

**개념**

- 서버에서 클라이언트를 인증하면, 서버는 세션을 생성하고 그 식별자인 `session ID`를 쿠키에 담아 클라이언트에 전달합니다.
- 이후 클라이언트는 자동으로 쿠키를 서버에 전송하여 인증을 계속 유지합니다.

**특징**

- 서버에 세션 정보 저장.
- 브라우저가 자동으로 쿠키를 포함시켜 전송.
- 보통 `HttpOnly`, `Secure`, `SameSite` 등의 옵션으로 보호 가능.

**장점**

- 쿠키는 자동으로 요청에 포함되므로 개발자가 따로 토큰을 넣을 필요 없음.
- `HttpOnly` 설정을 하면 **JavaScript에서 접근 불가**해 XSS에 어느 정도 강함.
- `SameSite` 옵션으로 CSRF 방지 가능.

**단점**

- CSRF 공격의 대상이 될 수 있음 (SameSite 설정 안 할 경우).
- 세션 데이터를 서버가 유지해야 하므로 **서버 메모리 부담** 존재.

### 🔐 2. 로컬 스토리지 + 토큰 방식 (ex. JWT)

**개념**
- 서버가 인증을 성공하면 **토큰(JWT 등)**을 클라이언트에 반환하고, 클라이언트는 이를 **로컬 스토리지**에 저장.
- 이후 요청 시, 클라이언트가 직접 `Authorization` 헤더에 토큰을 넣어 요청.

**특징**

- 서버는 세션을 따로 유지하지 않음 (stateless).
- 클라이언트 측 코드에서 토큰 관리.
- 일반적으로 토큰은 `Authorization: Bearer <token>` 형태로 전달됨.

**장점**

- 서버 확장에 유리 (stateless).
- REST API와 잘 어울림.
- 클라이언트에서 더 유연한 토큰 처리 가능.

**단점**

- 로컬 스토리지는 JavaScript에서 접근 가능하므로 **XSS 공격에 매우 취약**.
- 브라우저가 자동으로 토큰을 보내주지 않으므로, 매 요청마다 헤더에 수동으로 추가해야 함.
- 토큰 만료, 갱신 등의 처리를 클라이언트가 더 신경 써야 함.

### ✅ 어떤 걸 써야 할까?

- **전통적인 웹 애플리케이션** (예: 서버 렌더링, 로그인 페이지 있음) → **세션 쿠키** 적합
    
- **SPA + REST API 기반 프론트엔드** (React, Vue 등) → **로컬 스토리지 + JWT** 적합 (단, 보안 신경 써야 함)
    

> 단, **로컬 스토리지 대신 쿠키 + JWT 조합도 가능**합니다. 쿠키에 JWT를 넣고 `HttpOnly + Secure + SameSite=Strict`로 설정하면 보안이 강화됩니다.
